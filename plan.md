```md
# plan.md

## Project Goal

Build a **personal portfolio website** with a **Notion-style aesthetic** — clean, minimal, content-first, elegant typography, soft dividers, generous whitespace, subtle interactions, and a structured reading experience.

This should not feel like a flashy agency template.  
It should feel like a **serious developer workspace on the web**:
- minimal
- readable
- calm
- polished
- structured
- highly intentional

The site should present the developer as credible, thoughtful, and product-focused.

---

## Core Direction

Design and build a portfolio website inspired by the usability and rhythm of Notion:
- clean page structure
- muted colors
- strong typography hierarchy
- block-based layout
- subtle borders and dividers
- minimal visual noise
- smooth but restrained motion
- content-first presentation

The final result should feel like:
- **Notion meets modern developer portfolio**
- editorial and product-like
- interactive, but never loud
- premium through spacing, alignment, and clarity

---

## Design Principles

### 1. Content first
Every section must prioritize readability and hierarchy over decoration.

### 2. Minimal but not empty
The interface should feel refined, not unfinished.

### 3. Soft visual system
Use subtle shadows, light borders, muted backgrounds, and restrained contrast.

### 4. Structured like a document
The experience should feel like navigating a beautifully organized set of notes/pages.

### 5. Motion should support clarity
Animations should be gentle and purposeful:
- fade in
- slight upward reveal
- smooth hover transitions
- section transitions with low visual intensity

No flashy parallax, no aggressive gradients, no over-designed cards.

---

## Desired Visual Style

### Overall Feel
- Notion-inspired
- monochrome / neutral-first palette
- lightweight and elegant
- modern editorial layout
- calm developer aesthetic

### Typography
Use strong typography hierarchy similar to a writing/productivity tool:
- clean sans-serif for UI and body
- optional subtle serif accent only if it improves polish
- large, confident headings
- comfortable paragraph width
- readable line-height
- clear block spacing

### Color Palette
Prefer a restrained palette:
- off-white / warm white / soft gray background
- near-black primary text
- muted gray secondary text
- faint border tones
- one restrained accent color maximum

Avoid neon, heavy gradients, and loud saturated sections unless extremely controlled.

### Surfaces
- soft cards or block sections
- thin borders
- rounded corners used sparingly
- clean separators
- subtle hover states

---

## Website Structure

## 1. Hero Section
Purpose: immediate identity and first impression.

Include:
- name
- role/title
- short one-line intro
- short supporting paragraph
- primary CTA
- secondary CTA
- optional availability/status badge

Tone:
- direct
- calm
- credible
- not overhyped

Possible content direction:
- “Frontend Developer crafting thoughtful web experiences.”
- “I build clean, scalable, user-focused digital products.”

Hero should feel like the top of a Notion page:
- strong heading
- concise description
- well-spaced actions
- maybe a small icon/avatar/block marker

---

## 2. About Section
Purpose: give context beyond the title.

Include:
- who the developer is
- what they focus on
- what kind of work they enjoy
- their current direction / ambition

This should read like a polished personal note, not a generic bio.

Style:
- 2–4 compact paragraphs
- high readability
- honest and grounded
- not corporate

---

## 3. Skills / Toolbox Section
Purpose: quickly show technical strengths.

Structure skills like categorized Notion blocks.

Suggested categories:
- Languages
- Frontend
- Backend
- Tools
- Design / Workflow
- Currently Exploring

Presentation options:
- compact grid
- grouped lists
- tag-like blocks
- mini columns

Should feel organized and scan-friendly, not like keyword stuffing.

---

## 4. Featured Projects Section
Purpose: this is the main proof-of-work section.

Each project should include:
- project name
- short description
- problem or purpose
- stack used
- role/contribution
- impact/outcome
- GitHub link
- live demo link
- optional screenshot/mockup

Project cards/blocks should feel like clean document entries or linked database rows, not loud marketing tiles.

Each project must answer:
- what it is
- why it matters
- what was built
- what makes it interesting

Highlight 3–6 strongest projects only.

---

## 5. Experience Section
Purpose: show practical work and credibility.

Include:
- internships
- freelance work
- shipped client work
- major collaborations

Each item should contain:
- role
- company/client
- duration
- summary
- key contributions
- measurable outcomes if available

Should look like a timeline or stacked document entries.

---

## 6. Achievements / Certifications Section
Purpose: support credibility without overpowering the portfolio.

Include only meaningful items:
- hackathons
- competition rankings
- scholarships
- certifications relevant to target role
- open-source milestones
- speaking / community contributions if applicable

Keep concise and curated.

---

## 7. Education Section
Purpose: give formal background.

Include:
- degree
- institution
- years
- relevant coursework if truly useful
- notable academic work if relevant

Keep this section lighter than projects.

---

## 8. Contact Section
Purpose: make outreach easy.

Include:
- email
- GitHub
- LinkedIn
- resume link
- portfolio source code if desired

Can also include a short closing note like:
- open to internships
- open to freelance
- available for collaboration

This section should feel clean and intentional, not like footer filler.

---

## Optional Sections

### Writing / Notes
A Notion-style portfolio can strongly benefit from a writing section:
- thoughts on frontend
- UI notes
- engineering lessons
- project breakdowns

This adds depth and matches the Notion inspiration well.

### Uses / Setup
A page showing tools, workflow, editor, setup, and favorite stack.

### Now Page
A simple “what I’m currently focused on” section/page.

### GitHub / Activity Snapshot
Optional if integrated tastefully and minimally.

---

## Layout & UX Notes

### Navigation
Keep navigation minimal:
- Home
- Projects
- About
- Contact
- optional Notes

Sticky top nav is okay if subtle.

### Scrolling Experience
Single-page layout is preferred unless content grows large.  
Scrolling should feel smooth and document-like.

### Grid System
Use a restrained layout:
- max width content container
- generous padding
- clear vertical rhythm
- project blocks can break uniformity slightly for visual interest

### Responsiveness
Mobile experience must feel equally polished:
- stacked layout
- preserved typography hierarchy
- no cramped grids
- tap-friendly spacing

---

## Motion Direction

Motion should be subtle and premium.

Use:
- fade-in on section reveal
- gentle translate-y on entry
- soft hover transitions
- understated button interactions
- smooth anchor scrolling

Avoid:
- large springy motion
- flashy scroll effects
- over-animated cards
- distracting loaders

The portfolio should feel calm and composed.

---

- SectionHeader

- InlineLink

- MinimalButton
- Divider

- low-noise
- content-led
- soft borders
- subtle states
- strong spacing

## Technical Expectations

The site should be:
- fast
- fully responsive
- accessible
- SEO-friendly
- easy to update
- cleanly structured

Recommended qualities:
- semantic HTML
- reusable component structure
- strong spacing system
- optimized fonts/images
- dark mode optional if executed cleanly

## Content Expectations

The copy must feel:
- sharp
- human
- confident
- concise
- not AI-generic
- not overly formal
- not buzzword-heavy

Avoid lines that sound vague such as:
- “I am passionate about technology”
- “I love solving problems”
- “I am a hardworking individual”

Prefer specific and grounded writing.

- prioritize clarity and structure over visual excess

**“A strong developer portfolio designed with the calm clarity of Notion.”**

## Final Build Priorities

Priority order:
1. typography

3. content hierarchy
4. project presentation
5. subtle interactions
6. responsiveness

- clarity over decoration
- structure over gimmicks
- restraint over excess
- readability over visual noise
```
